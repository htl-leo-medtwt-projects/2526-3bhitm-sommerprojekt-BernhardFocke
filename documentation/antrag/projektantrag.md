# Spinnig Records – Semesterprojekt MEDT 25/26

# USP

### Grundidee
Spinnig Records soll eine interaktive Website mit Fokus darauf sein, das Erlebnis Musik zu hören deutlich zu verstärken und eine Oberfläche zu erstellen, die sich bewusst von typischen Musikstreamingplattformen abhebt.

Der Fokus liegt darauf, dem User ein möglichst intensives und personalisierbares Hörerlebnis zu bieten, das über das reine Abspielen von Musik hinausgeht. Während viele Streamingdienste primär auf große Musikbibliotheken und Algorithmen setzen, konzentriert sich Spinnig Records stärker auf das **aktive Musikerlebnis, Individualisierung und Interaktivität**.

Die Plattform soll Nutzer dazu einladen, Musik bewusster zu erleben, ihre Hörgewohnheiten zu analysieren und neue Musik auf eine ganz neue Art zu entdecken.

---

# UI & UX

## Funktionen

### Music Player
Der Music Player bildet das zentrale Element der Website.

Er soll ein **eigenständiges, individuelles Design** besitzen und deutlich interaktiver sein als klassische Player.

Mögliche Funktionen:

- Anpassung der **Abspielgeschwindigkeit**
- **Designanpassungen** durch den User (Farbschema, Hintergrund)
- intuitives und minimalistisches Bedienkonzept
- visuelle Elemente, die auf die Musik reagieren

Der Player soll dabei sowohl funktional als auch ästhetisch überzeugen.

---

### Tracker
Der integrierte Tracker analysiert das Hörverhalten der Nutzer und stellt dieses übersichtlich dar.

Mögliche Statistiken:

- aktuelle Lieblingssongs
- meistgehörte Songs
- gesamte Hörzeit
- Hörzeit pro Tag oder Woche
- Lieblingsgenres

Die Daten werden in einem übersichtlichen Dashboard dargestellt und sind jederzeit abrufbar.

---

### Lieblingssongs und Playlists
Nutzer können Songs speichern und eigene Playlists erstellen.

Funktionen:

- Songs zu Favoriten hinzufügen
- eigene Playlists erstellen
- Songs zu bestehenden Playlists hinzufügen
- Playlist-Reihenfolge verändern

Dies ermöglicht eine persönliche Organisation der Lieblingsmusik.

# Erweiterte Funktionen

## Social Features
Spinnig Records soll auch soziale Elemente enthalten, die Interaktion zwischen Nutzern ermöglichen.

Mögliche Funktionen:

### Song Reactions
User können Songs mit Reaktionen versehen, zum Beispiel:

- Fire
- Chill
- Like  

So können Songs schnell bewertet und hervorgehoben werden.

---

### Shareable Listening Cards
Wenn ein Nutzer einen Song hört, kann automatisch eine **sharebare Karte** generiert werden.

Diese zeigt zum Beispiel:

- aktuell gespielter Song
- Künstler
- Albumcover
- Benutzername

Diese Karte kann als Bild exportiert oder geteilt werden.

---

## Discovery Features
Ein weiteres Ziel der Plattform ist es, Nutzern zu helfen **neue Musik zu entdecken**.

### Random Discovery
Ein Button wie **„Überrasch mich“** spielt einen zufälligen Song aus der Datenbank ab.

Optional kann der Nutzer Filter setzen, z.B.:

- Genre
- Stimmung
---

### Hidden Gems
Das System kann Songs hervorheben, die:

- selten gehört werden
- aber gute Bewertungen haben

So können weniger bekannte Songs entdeckt werden.

---

### Song of the Day
Jeden Tag wird ein **„Song des Tages“** hervorgehoben.

Dieser kann automatisch aus der Datenbank ausgewählt werden und wird auf der Startseite angezeigt.

---

# Besondere Player Features

## Vinyl Mode
Ein besonderes Feature des Music Players ist der **Vinyl Mode**.

In diesem Modus wird der Player visuell wie ein **Plattenspieler** dargestellt.

Mögliche Elemente:

- rotierende Schallplatte
- Tonarm-Animation
- Albumcover als Label auf der Platte
- realistische Start- und Stop-Animation

Der Vinyl Mode soll vor allem das **nostalgische Gefühl des Musik-Hörens** verstärken und ein einzigartiges visuelles Erlebnis schaffen.

Dieses Feature hebt sich bewusst von klassischen Streaming-Playern ab und trägt zum besonderen Charakter der Plattform bei.
